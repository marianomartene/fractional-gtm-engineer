# Contributing

Thanks for wanting to improve this list. A few ground rules before you open a PR.

## What belongs here

- Real, useful, verifiable resources: people, tools, playbooks, communities, and
  case studies related to fractional GTM and GTM engineering.
- Fair inclusion of competitors and peers. If it's genuinely useful to someone
  hiring or working in this space, it belongs, even if it competes with the
  maintainer.
- One entry per line, in the format:
  `[Name](url) - short, honest description.`

## What doesn't

- Thin SEO content, listicles with no real information, or pages that exist only
  to capture leads.
- Link farms: if an entry doesn't teach or help someone evaluating fractional
  GTM, don't add it.
- Dead links, or links you haven't personally checked still load.
- Self-promotion with no substance behind it. A one-line personal site with no
  public track record isn't enough; a public LinkedIn profile that clearly
  self-identifies as fractional GTM/RevOps, a real Substack, or a real GitHub
  repo, is.
- Pricing or rate information for any person or company listed. This list
  describes what things are, not what they cost.

## How to submit

1. Fork the repo and add your entry to the relevant section, alphabetized
   within that section.
2. Keep the description to one line, plain and factual, no marketing copy.
3. Open a PR explaining briefly why the resource belongs (a link to the
   resource itself is usually enough).
4. Maintainers will check the link works and that the entry is real before
   merging. Expect some entries to be declined if they can't be verified.

If you're adding yourself as an operator/practitioner: link only to a public
profile or site that clearly self-identifies as fractional GTM engineering (or
a clearly adjacent fractional GTM/RevOps role), and keep the description
factual.
